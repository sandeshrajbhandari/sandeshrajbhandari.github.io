Build notes

How these prompt libraries were built

This master repo contains two static prompt libraries that can be copied into a GitHub personal site: one for ERNIE image prompts and one for Z-Image Turbo prompts. The folders stay separate, but the interface, filtering, search, modal previews, and copy behavior now follow the same structure.

Source Data

ERNIE prompts

Images and prompt text were scraped directly from ernieimageprompt.com. The revised prompt is treated as the primary prompt because it is the version used for image generation.

Z-Image prompts

Images came from the ModelScope Z-Image Gallery. That gallery did not include prompts, so the prompts were generated locally from the gallery images.

Z-Image Prompt Generation

The Z-Image prompts were generated on a local laptop using a Qwen3.5 9B GGUF model through llama.cpp. The machine used a 16GB RTX 3080, which made it practical to caption and rewrite the gallery images into usable generation prompts without depending on hosted prompt text.

The local WebP images are optional in the Z-Image folder. Each prompt also stores the original image URL as a fallback, so the static gallery can still load images on a GitHub personal site without uploading the scraped Z-Image picture files.

Classification

Category and tag labels

Both libraries were classified so the previews can be searched and filtered. ERNIE uses category filters; Z-Image uses categories plus tags because the generated prompts benefit from more granular browsing.

Modal classifier workflow

Classification was iterated with a zero-shot prompt classifier running on Modal. Long prompts were bucketed by token length and processed without truncating prompt text, which improved the Z-Image labels compared with shorter fixed-length classification passes.

Modal made the classifier loop fast enough to repeat while tuning candidate labels, reviewing weak buckets, and regenerating the static `prompt_categories.js` files. Cursor's coding agent was used to speed up the iteration cycle around scripts, UI changes, and classification reviews.

Static Library Shape

  • Each prompt library has its own folder with `index.html`, `styles.css`, `app.js`, prompt data, and classifications.
  • Both previews have a compact header, search, filters, randomized default sorting with a static-sort switch, grid copy buttons, and modal prompt copy.
  • ERNIE copy actions use the revised prompt first; other prompt versions remain available inside modal tabs.
  • Z-Image cards can load local WebP files when present and fall back to the original hosted image URLs when they are not.

The old standalone repositories are archived. New updates are collected in the prompt-library-master repo.
